# Architecture Decision Records

ADRs capture significant, hard-to-reverse design choices. Each ADR records the
context, the decision itself, alternatives considered, and the consequences.

## Naming

ADR files follow the pattern `ADR-NNN-short-slug.md` where `NNN` is a
zero-padded sequential number (e.g. `ADR-001-terminusdb-as-primary-store.md`).

## Statuses

| Status | Meaning |
|---|---|
| **Proposed** | Written but not yet agreed upon |
| **Accepted** | Agreed and in effect |
| **Superseded** | Replaced by a later ADR (the superseding ADR is noted in the text) |
